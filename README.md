# PhaDataPro AI Chat Assistant

**Многомодульный русскоязычный чат-бот** для системы PhaDataPro на базе **Streamlit** и **OpenAI GPT-4o**.

Ассистент отвечает на вопросы о работе с аналитическими отчётами, поддерживает **все модули и каналы**, синонимизирует ввод пользователя, сохраняет контекст между вопросами и даёт **пошаговые инструкции**.

---

## Что умеет бот

### Поддерживаемые модули и каналы

**Аптечные данные** — мультивыбор каналов:
| Канал | Описание |
|-------|----------|
| Sell Out | Розничные продажи конечным пациентам через аптеки |
| Sell In | Оптовые закупки аптечными сетями у дистрибьюторов |
| Sell Out + E-com | Комбинированный: розница + онлайн-продажи |
| Перемещение | Передача товара между складами и филиалами |
| Остатки | Текущее наличие товара в аптеках |
| E-com | Продажи через интернет-аптеки |

**Данные дистрибьютора** — мультивыбор каналов:
| Канал | Описание |
|-------|----------|
| Sell In | Закупки дистрибьютора у производителя |
| Sell In + Tender | Комбинированный: закупки + госзакупки |
| Tender | Государственные тендерные закупки |
| Остатки | Наличие товара на складах дистрибьютора |
| Импорт | Ввоз товара в страну (международные поставки) |

### Поддерживаемые типы анализа

| Анализ | Ключевые показатели |
|--------|-------------------|
| Продажи в разрезе регионов / АС / менеджеров / дистрибьюторов / филиалов | Упаковки, Сумма |
| Сравнение плана с фактом (АС и ДС) | % выполнения плана |
| Анализ процента выполнения плана | Зелёная/жёлтая/красная зоны |
| Анализ прироста продаж по упаковкам и суммам | Прирост упаковок (%), Прирост суммы (%) |
| Анализ доли по выбранным данным | Доля упаковок (%), Доля суммы (%) |
| Высокопотенциальные точки АС (Категория СПА) | Потенциал vs факт |
| Категоризация аптек по продажам (ABC-анализ) | Категория A / B / C / D |

### Параметры отчётов

- **Выгрузка:** по Упаковкам или по Суммам (CIP / Оптовая цена)
- **4 валюты:** KZT (тенге), USD (доллар), EUR (евро), RUB (рубль)
- **Временные отрезки:** Декада (10 дней), Месяц, Квартал, Год
- **Вид цены:** CIP (с логистикой) или Оптовая (без логистики)
- **Факт/План:** Факт, План, Факт + План

### Интеллектуальные возможности

- **Синонимизация** — понимает все варианты написания на русском и латинице
- **Сохранение контекста** — помнит выбранный модуль и канал на протяжении всего диалога
- **Уточнение** — если вопрос неясен, просит указать модуль/канал
- **Напоминание о периоде** — всегда напоминает, что период обязателен
- **Простые вопросы** — отвечает на вопросы о входе в систему, навигации, параметрах

## Deploy to Streamlit Community Cloud

1. Загрузите проект в GitHub-репозиторий (убедитесь, что `.gitignore` включает `.streamlit/secrets.toml`).
2. Перейдите на [share.streamlit.io](https://share.streamlit.io) и нажмите **New app**.
3. Укажите репозиторий и **Main file path**:
   ```
   streamlit_app.py
   ```
4. Перейдите в **App settings → Secrets** и добавьте:
   ```toml
   OPENAI_API_KEY = "ваш_ключ_здесь"
   ```
5. Нажмите **Deploy** — приложение запустится автоматически.

> ⚠️ **Никогда не вставляйте реальный API key в код или в файлы, которые попадают в Git.**
> Файл `.streamlit/secrets.toml` добавлен в `.gitignore` и не будет опубликован.

## Быстрый старт (локально)

### 1. Установка

```bash
cd "чат бот маршрутизатор"

python -m venv .venv

# Windows
.venv\Scripts\activate

# macOS/Linux
source .venv/bin/activate

pip install -r requirements.txt
```

### 2. API Key для локального запуска

**Вариант A** — через `.streamlit/secrets.toml` (рекомендуется):
```toml
OPENAI_API_KEY = "ваш_ключ_здесь"
```

**Вариант B** — через переменную окружения:
```bash
# Windows PowerShell
$env:OPENAI_API_KEY = "ваш_ключ_здесь"

# macOS/Linux
export OPENAI_API_KEY="ваш_ключ_здесь"
```

### 3. Запуск

```bash
streamlit run streamlit_app.py
```

Откроется в браузере: `http://localhost:8501`

## Как работает синонимизация

Пользователь пишет в любом формате — система приводит к единому ключу:

| Что написал пользователь | Нормализованный ключ |
|--------------------------|---------------------|
| сел ин / селл ин / sell in / закупки / опт | `sell_in` |
| сел аут / селл аут / sell out / розница | `sell_out` |
| sell out + e-com / аут+ком / розница+онлайн | `sell_out_ecom` |
| перемещение / movement / transfer / логистика | `placement` |
| остатки / stock / наличие / запасы / inventory | `stock` |
| e-com / ecom / онлайн / интернет-аптека | `ecom` |
| тендер / госзакупка / гос. закупка / tender | `tender` |
| импорт / import / ввоз / иностранный товар | `import_channel` |

## Как работает контекст

Контекст сохраняется между сообщениями:

```
Пользователь: "Аптечные данные"
Бот: [устанавливает модуль pharmacy_data]

Пользователь: "сел ин"
Бот: [понимает — Sell In в аптечных данных] → пошаговая инструкция

Пользователь: "Как посмотреть прирост?"
Бот: [помнит Sell In, аптечные] → инструкция по приросту для Sell In
```

При неоднозначности (Sell In и Остатки есть в обоих модулях) — бот уточняет:

```
Пользователь: "Sell In"  [без указания модуля]
Бот: "Уточните: Аптечные данные или Данные дистрибьютора?"
```

## Конфигурация

### Основные параметры (streamlit_app.py)

```python
COMPLETION_MODEL    = 'gpt-4o'               # Модель для ответов
EMBEDDING_MODEL     = 'text-embedding-3-small'  # Модель для поиска
MAX_TOKENS          = 2000                   # Макс. токенов в ответе
RELEVANCE_THRESHOLD = 0.12                   # Порог релевантности поиска
```

### OpenAI API Key

Ключ загружается автоматически из двух источников (в порядке приоритета):
1. **Streamlit Secrets** — `st.secrets["OPENAI_API_KEY"]` (используется на Streamlit Cloud)
2. **Переменная окружения** — `os.getenv("OPENAI_API_KEY")` (для локального запуска)

Ключ **никогда** не хранится в коде и не попадает в Git.

## Логирование

**Лог запросов** (`logs/chat_history.log`) — JSONL формат:
```json
{"timestamp": "...", "user_input": "...", "context": {...}, "response": "...", "sources": [...]}
```

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Отчёт пустой / данные не загружаются | Выберите **Период** — он обязателен |
| «Документы базы знаний не найдены» | Убедитесь, что файлы БЗ в корневой папке |
| «Превышен лимит запросов» | Подождите 1-2 минуты и повторите |
| «Неверный API ключ» | Проверьте ключ в боковой панели |
| Бот не понимает синоним | Попробуйте: `sell in`, `тендер`, `импорт`, `остатки` |

## Архитектура

```
Frontend:  Streamlit (st.chat_message — нативный рендеринг markdown)
Backend:   Python + requests (прямые вызовы OpenAI API)
AI:        OpenAI GPT-4o (ответы) + text-embedding-3-small (семантический поиск)
Поиск:     Embeddings + косинусное сходство + контекстные бонусы
Данные:    JSON (история), JSONL (логи), MD/DOCX/PDF (база знаний)
Контекст:  Streamlit session_state (модуль + канал + тип анализа)
Синонимы:  Словарь с нормализацией кириллица/латиница + контекстное разрешение омонимии
```
